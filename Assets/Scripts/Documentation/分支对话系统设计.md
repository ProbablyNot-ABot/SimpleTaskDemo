# 分支对话系统设计文档（简化版）

## 📖 概述

这是一个支持**分支结构**和**任务挂钩**的简洁对话系统。

---

## 🏗️ 数据结构

### 核心类

```
DialogData (ScriptableObject)
    ├── dialogId: string           // 对话唯一标识
    ├── npcName: string            // NPC 名称
    ├── nodes: List<DialogNode>    // 所有对话节点
    ├── startNodeId: string        // 起始节点 ID
    └── allowRepeat: bool          // 是否允许重复
    
DialogNode (对话节点)
    ├── nodeId: string             // 节点唯一标识
    ├── speakerName: string        // 说话者
    ├── content: List<string>      // 对话内容（多页）
    ├── options: List<DialogOption>// 选项列表
    ├── enterCondition             // 进入条件（任务状态）
    └── description                // 节点说明

DialogOption (对话选项)
    ├── optionText: string         // 选项文字
    ├── nextNodeId: string         // 跳转节点 ID
    ├── condition                  // 选择条件（任务状态）
    └── action                     // 选择后的动作（任务相关）

DialogCondition (对话条件)
    ├── hasCondition: bool         // 是否有条件
    ├── questId: string            // 任务 ID
    └── requiredState: QuestState  // 任务状态

DialogAction (对话动作)
    ├── hasAction: bool            // 是否有动作
    ├── type: ActionType           // 动作类型
    └── questId: string            // 任务 ID

QuestState (任务状态)
    ├── NotStarted                 // 未开始
    ├── InProgress                 // 进行中
    ├── Completed                  // 已完成
    └── Failed                     // 失败
```

---

## 🎯 任务状态（QuestState）

| 状态 | 说明 | 使用场景 |
|------|------|----------|
| `NotStarted` | 未开始 | 任务还没接取 |
| `InProgress` | 进行中 | 任务已接取但未完成 |
| `Completed` | 已完成 | 任务已完成 |
| `Failed` | 失败 | 任务失败 |

---

## 🎬 动作类型（DialogAction）

| 类型 | 说明 | 使用场景 |
|------|------|----------|
| `None` | 无动作 | 纯对话 |
| `StartQuest` | 开始任务 | 接取任务 |
| `CompleteQuest` | 完成任务 | 交付任务 |

## 🔄 流程图

```
玩家点击 NPC
    ↓
DialogManager.StartDialog(dialogData)
    ↓
获取 startNodeId
    ↓
找到对应 DialogNode
    ↓
检查 enterCondition (如果有)
    ├─ 不满足 → 使用 repeatNodeId 或结束
    └─ 满足 → 继续
        ↓
显示 content[0], content[1]... (多页)
    ↓
玩家点击下一步
    ↓
遍历 options:
    对每个 option:
        ├─ 检查 condition (如果有)
        │   ├─ 不满足 → 隐藏/禁用
        │   └─ 满足 → 显示
        └─ 创建按钮
            ↓
玩家选择选项
    ↓
执行 option.action (如果有)
    ├─ StartQuest → 开始任务
    └─ CompleteQuest → 完成任务
    ↓
跳转到 option.nextNodeId
    ↓
检查 nextNodeId:
    ├─ 不为空 → 找到节点，循环
    └─ 为空 → 结束对话
```

## 🛠️ 实现要点

### 1. 条件检查

```csharp
public static class DialogConditionChecker
{
    public static bool CheckCondition(DialogCondition condition)
    {
        if (!condition.hasCondition)
            return true;
        
        // 检查任务状态
        QuestState currentState = QuestManager.Instance.GetQuestState(condition.questId);
        return currentState == condition.requiredState;
    }
}
```

---

### 2. 动作执行

```csharp
public static class DialogActionExecutor
{
    public static void ExecuteAction(DialogAction action)
    {
        if (!action.hasAction)
            return;
        
        switch (action.type)
        {
            case DialogAction.ActionType.StartQuest:
                QuestManager.Instance.StartQuest(action.questId);
                break;
            
            case DialogAction.ActionType.CompleteQuest:
                QuestManager.Instance.CompleteQuest(action.questId);
                break;
        }
    }
}
```

## 📊 数据验证

### 编辑器验证

DialogData 自带 `Validate()` 方法，可以检查：

- ✅ dialogId 是否为空
- ✅ startNodeId 是否为空
- ✅ 是否有至少一个节点
- ✅ 节点 ID 是否重复
- ✅ 起始节点是否存在
- ✅ 选项跳转的节点是否存在

使用方式：

```csharp
#if UNITY_EDITOR
List<string> errors = dialogData.Validate();
if (errors.Count > 0)
{
    foreach (var error in errors)
    {
        Debug.LogError(error);
    }
}
#endif
```
